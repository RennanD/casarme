# Copy Structure Reference

Detailed requirements for each landing page section.

## 1. Hero Section

### Requirements
- **Title**: ≤70 characters, must communicate category + differentiator
- **Subtitle**: ≤150 characters, expands on the value proposition
- **CTA**: ≤20 characters, action-oriented
- **Image**: Specify what visual represents the product

### Title Formula
Answer two questions:
1. What is this? (category)
2. Why is it different? (key differentiator)

**Examples:**
- "O primeiro CRM que preenche dados sozinho" (CRM + auto-fill differentiator)
- "Meeting notes that write themselves" (Meeting tool + AI differentiator)

### Subtitle Formula
Expand on the HOW and WHO:
- How the differentiator works
- Who benefits most

### Hero Image Spec
Include:
- What to show (product screenshot, mockup, illustration)
- Key elements that should be visible
- Mood/style aligned with design system

## 2. Social Proof Section

### Options (pick one or combine)
1. **User count**: "Usado por mais de X.XXX usuários"
2. **Company logos**: "Empresas como [Logo1], [Logo2], [Logo3] já usam"
3. **Rating**: "Avaliação 4.9 ★★★★★"
4. **Certification**: Relevant trust badges

### Guidelines
- Use real numbers if available
- If no data, suggest realistic placeholder with note to update
- Logos should be recognizable to target audience

---

## 3. Product vs. Alternatives Section

### Structure
Compare the product against top 2-3 alternatives identified in positioning.

### Format Options

**Option A: Table comparison**
| Feature | [Product] | [Alternative 1] | [Alternative 2] |
|---------|-----------|-----------------|-----------------|
| [Unique feature 1] | ✓ | ✗ | ✗ |
| [Unique feature 2] | ✓ | Partial | ✗ |

**Option B: Before/After narrative**
- "Antes: [pain with alternatives]"
- "Depois: [benefit with product]"

**Option C: Side-by-side cards**
- Card 1: "Com [Alternative]" - list pain points
- Card 2: "Com [Product]" - list benefits

### Guidelines
- Focus ONLY on differentiators
- Be factual, not aggressive
- Show clear winner without being negative

---

## 4. Features & Value Section

### Structure
For each unique feature (3-5 features max):
- **Title**: Feature name (≤40 chars)
- **Description**: Benefit explanation (≤100 chars)
- **Visual**: Screenshot or illustration spec

### Guidelines
- ONLY include features that competitors don't have
- Table stakes features are implied by the category
- Each feature must map to a value theme from positioning
- Order by importance to target customer

### Visual Spec Format
```
[Generate] or [Screenshot needed]
- Screen: [which product screen to show]
- Highlight: [which element to emphasize]
- Context: [what user action is being shown]
```

---

## 5. How It Works Section

### Structure
3-5 steps maximum, showing the user journey:

**Step format:**
- **Number**: 1, 2, 3...
- **Title**: Action verb + object (≤30 chars)
- **Description**: What happens (≤80 chars)
- **Visual**: Icon or mini-screenshot

### Guidelines
- Start from user's first action
- End with achieved outcome
- Keep steps simple and sequential
- Avoid technical jargon

---

## 6. Testimonials Section

### Structure
3-4 testimonials that reinforce competitive claims.

**Testimonial format:**
```
"[Quote that mentions specific differentiator or comparison to alternative]"
— [Name], [Role that matches ICP], [Company type if relevant]
```

### Guidelines for Invented Testimonials
- Role must match the target customer profile from positioning
- Quote must reinforce a specific competitive claim
- Reference the alternative they stopped using
- Sound natural, not marketing-speak

**Example:**
```
"Depois que conheci o [Product], nunca mais voltei para planilhas. 
O que levava 2 horas agora leva 5 minutos."
— Marina Costa, Head de Operações, Startup de Logística
```

### Note to User
Mark invented testimonials clearly:
```
[SUGESTÃO - substituir por depoimento real se disponível]
```

## 7. Target Customers Section

### Structure
3-4 cards showing ideal customer profiles.

**Card format:**
- **Persona title**: Role or type (e.g., "Product Managers", "Solo Founders")
- **Description**: Why this product is perfect for them (≤80 chars)
- **Icon or illustration**: Visual representation

### Guidelines
- Based on target customer characteristics from positioning
- Show the BEHAVIOR or SITUATION that makes them ideal
- Not demographics, but psychographics

---

## 8. CTA Section

### Structure
- **Headline**: Emotional/aspirational (≤60 chars)
- **Supporting points**: 2-3 bullet points addressing objections
- **CTA button**: Same as hero CTA
- **Trust element**: Risk reducer (free trial, money-back, etc.)

### Guidelines
- Headline should paint the desired outcome
- Bullet points remove friction (time, risk, complexity)
- CTA should be identical to hero for consistency

---

## 9. FAQ Section

### Structure
5-7 questions that address common objections.

**Categories to cover:**
1. **Price/Value**: "How much does it cost?" / "Is there a free trial?"
2. **Trust/Security**: "Is my data safe?" / "Who else uses this?"
3. **Effort/Time**: "How long to set up?" / "Do I need technical skills?"
4. **Comparison**: "How is this different from [Alternative]?"
5. **Support**: "What if I need help?"

### Guidelines
- Questions should be in customer's voice
- Answers should be concise and confident
- Use objections identified in positioning analysis
- End answers with subtle CTAs when appropriate

---

## 10. Footer Section

### Standard Elements
- Logo
- Navigation links (Features, Pricing, About, Contact)
- Legal links (Privacy Policy, Terms of Service)
- Social media links (if applicable)
- Trust badges/certifications
- Copyright notice

### Guidelines
- Keep minimal and clean
- Include only essential links
- Match design system colors
